# Engineering Setup — 工程标配文件体系

> 一套轻量级、可复用的工程文件规范，让每个项目在诞生之初就具备清晰的骨架。

无论你是一个人单干、多 Agent 协作，还是团队开发，这套文件体系帮助你做到：

- **有动作必记录** — 做了就有痕迹，不等老板提醒
- **遇坑必记** — 踩过的坑变成团队的资产，不会反复踩
- **任务可追踪** — 条件具备直接执行，不需要决策的事项不问
- **新成员秒上手** — ENTRY.md 是第一站，5 分钟了解全局

## 文件体系

```
项目根目录/
├── ENTRY.md           ← 接入手则：新人/新Agent的第一站
├── work_log.md        ← 工作记录：有动作必记录
├── pitfalls.md        ← 踩坑记录：遇坑必记
├── tasks.md           ← 任务清单：条件具备直接执行
├── standards.md       ← 操作规范（按需创建）
├── report.md          ← 项目评估报告（里程碑时产出）
└── README.md          ← 工程概述
```

## 快速开始

```bash
# 1. 在项目根目录创建标配文件
touch ENTRY.md work_log.md tasks.md pitfalls.md README.md

# 2. 有规范需求时再建 standards.md
# 3. 阶段性里程碑时建 report.md

# 4. 用模板内容替换 {占位符}
```

## 核心理念

| 规则 | 说明 |
|------|------|
| **先看 ENTRY.md** | 新成员接入的第一站，5 分钟了解全局 |
| **有动作必记录** | 做完→记录→更新任务→继续。活干了不记=没干完 |
| **遇坑必记** | 现象+根因+解决+预防。遇坑不记=还会再踩 |
| **条件具备直接执行** | 不需要决策的事项直接动手，做了再记 |
| **记录是做完的一部分** | 完成=码写了+记录了+任务单更新了 |

## 文件模板说明

### ENTRY.md
项目首页。回答三个问题：这是什么项目？我在哪？下一步该干什么？

### work_log.md
**核心文件。** agent 的执行契约，不是日志。标准格式：
```
YYYY-MM-DD HH:MM | 执行人 | 做了什么
结果数据
- ✅ / ⚠️ / ❌ 结果
- 关联 pitfalls: #编号
- 关联 tasks: #编号
```

### pitfalls.md
所有踩过的坑。格式：日期→问题→根因→解决→预防。
目的是让同一坑不踩第二次。

### tasks.md
当前任务清单。每个任务含状态、负责人、优先级、验收标准。
完成标准：任务打勾 + work_log 有记录 = 真正做完。

### standards.md（按需）
项目特定的规范：路径规范、代码规范、安全规范、数据库规范。
**有规范才建，没有就不建。**

### report.md（里程碑时产出）
项目阶段性评估：架构、负责人、工作量分布、技术债务。
帮助新加入的人快速了解全局。

## 通用踩坑预防

| 类别 | 坑 | 预防 |
|------|----|------|
| 文件操作 | 大 JSON 写入半截损坏 | compact 格式 + batch 写入 + 原子替换 |
| 文件操作 | 多 Agent 并发写 | 写操作走队列，单一 agent 合并 |
| 文件操作 | 恢复错旧备份 | 恢复前检查备份的覆盖状态 |
| 代码 | 路径硬编码 | 全部用相对路径 Path(file).resolve().parent |
| 代码 | 幂等检查遗漏 | 遍历数组精确匹配，不靠顶层字段 |
| 代码 | 裸 except: | 全部用 except Exception: |
| 代码 | eval() 安全风险 | 用 ast.literal_eval() |
| 项目管理 | 活干了不记 | 完成标准=代码+记录+任务单 |
| 项目管理 | 踩坑不记 | 遇坑立刻记 pitfalls |
| 项目管理 | 等指令浪费时间 | 条件具备直接动手 |
| 跨 Agent | 覆盖他人数据 | 写走队列，不直写公共数据 |
| 跨 Agent | 不知道其他人在做什么 | work_log 标注执行人，tasks 标注负责人 |

## 使用场景

- **个人项目** — 防止自己忘记做到哪了
- **多 Agent 协作**（AI Agent） — 多个 AI agent 分工，必须有统一的记录和任务体系
- **团队开发** — 新成员迅速上手，遇坑经验不丢失
